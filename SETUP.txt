================================================================================
  PART B — BUILD & PUSH DOCKER IMAGE
================================================================================

  This section runs on YOUR LOCAL PC (not EC2).
  You need Docker Desktop installed: https://www.docker.com/products/docker-desktop/
  You need a Docker Hub account (free): https://hub.docker.com

  Docker Hub gives 1 free public image (unlimited pulls) or 1 private image.
  This project's image is safe to make public — no secrets are baked in
  (API keys and .env are injected at runtime via environment variables).


------------------------------------------------------------
  B1. LOG IN TO DOCKER HUB
------------------------------------------------------------

    docker login

  Enter your Docker Hub username and password.


------------------------------------------------------------
  B2. BUILD THE IMAGE
------------------------------------------------------------

    cd Portfolio/backend/budget-tracker

    docker build -t YOUR_DOCKERHUB_USERNAME/spending-api:latest .

  Replace YOUR_DOCKERHUB_USERNAME with your actual Docker Hub username.
  First build takes 5-10 minutes (downloads Python base + installs packages).
  Subsequent builds are fast (Docker layer cache).


------------------------------------------------------------
  B3. TEST THE IMAGE LOCALLY
------------------------------------------------------------

    docker run --rm -p 8000:8000 \
      -e API_KEY=gsk_your_key \
      -e MODEL=meta-llama/llama-4-scout-17b-16e-instruct \
      -e BASE_URL=https://api.groq.com/openai/v1 \
      -e ALLOWED_ORIGINS=http://localhost:3000 \
      YOUR_DOCKERHUB_USERNAME/spending-api:latest

  Test: http://localhost:8000/health → {"status":"ok"}
  Stop: CTRL+C


------------------------------------------------------------
  B4. PUSH TO DOCKER HUB
------------------------------------------------------------

    docker push YOUR_DOCKERHUB_USERNAME/spending-api:latest

  First push takes 3-8 minutes (uploads ~600MB of layers).
  Subsequent pushes only upload changed layers (usually < 1 minute).

  Your image is now at:
    https://hub.docker.com/r/YOUR_DOCKERHUB_USERNAME/spending-api

  PART C — FREE AWS EC2 DEPLOYMENT
================================================================================

  ---------------------------------------------------------

  AFTER 12 MONTHS:   ~$8.50/month


------------------------------------------------------------
  C1. LAUNCH EC2 INSTANCE
------------------------------------------------------------

  1. https://console.aws.amazon.com → EC2 → Launch Instance

  2. Settings:
       Name:             portfolio-server
       AMI:              Ubuntu Server 22.04 LTS (64-bit x86) — Free tier eligible
       Instance type:    t2.micro      ← free tier
       Key pair:         Create new → Name: portfolio-key → RSA → .pem
                         DOWNLOAD AND SAVE. Cannot re-download.

  3. Network settings → Edit → Create security group: portfolio-sg
       SSH    port 22   Source: My IP
       HTTP   port 80   Source: 0.0.0.0/0
       HTTPS  port 443  Source: 0.0.0.0/0

  4. Storage: 15 GiB gp3

  5. Launch Instance. Wait for State = "running".


------------------------------------------------------------
  C2. ALLOCATE ELASTIC IP (fixed IP that survives restarts)
------------------------------------------------------------

  EC2 Console → Network & Security → Elastic IPs
  → Allocate Elastic IP address → Allocate
  → Select the new IP → Actions → Associate Elastic IP address
  → Instance: portfolio-server → Associate

  Note your fixed Elastic IP — use it everywhere below.

  NOTE: Free while attached to a running instance.
  If you stop the instance, either keep the IP associated
  or release it (EC2 → Elastic IPs → Release) to avoid charges.


------------------------------------------------------------
  C3. SSH INTO EC2
------------------------------------------------------------

  Windows (PowerShell):

    icacls "C:\path\to\portfolio-key.pem" /inheritance:r
    icacls "C:\path\to\portfolio-key.pem" /grant:r "$($env:USERNAME):(R)"
    ssh -i "C:\path\to\portfolio-key.pem" ubuntu@YOUR_ELASTIC_IP

  Mac/Linux:

    chmod 400 ~/Downloads/portfolio-key.pem
    ssh -i ~/Downloads/portfolio-key.pem ubuntu@YOUR_ELASTIC_IP

  All commands from here run ON THE EC2 SERVER.


------------------------------------------------------------
  C4. ADD SWAP (prevents OOM crashes on 1GB t2.micro)
------------------------------------------------------------

    sudo fallocate -l 2G /swapfile
    sudo chmod 600 /swapfile
    sudo mkswap /swapfile
    sudo swapon /swapfile
    echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
    free -h    # should show Swap: 2.0G


------------------------------------------------------------
  C5. INSTALL DOCKER AND NGINX
------------------------------------------------------------

    sudo apt update && sudo apt upgrade -y

    # Docker
    curl -fsSL https://get.docker.com | sh
    sudo usermod -aG docker ubuntu
    newgrp docker

    # Nginx + Certbot
    sudo apt install -y nginx certbot python3-certbot-nginx

    # Verify
    docker --version
    nginx -v


------------------------------------------------------------
  C6. RUN THE BACKEND CONTAINER
------------------------------------------------------------

    docker pull YOUR_DOCKERHUB_USERNAME/spending-api:latest

    docker run -d \
      --name spending-api \
      --restart unless-stopped \
      --memory="900m" \
      --memory-swap="2900m" \
      -p 127.0.0.1:8000:8000 \
      -e MODEL=meta-llama/llama-4-scout-17b-16e-instruct \
      -e BASE_URL=https://api.groq.com/openai/v1 \
      -e API_KEY=gsk_your_actual_groq_key \
      -e FILES_DIR=/tmp/spending-files \
      -e ALLOWED_ORIGINS=https://yourdomain.com,https://www.yourdomain.com \
      YOUR_DOCKERHUB_USERNAME/spending-api:latest

  Replace: YOUR_DOCKERHUB_USERNAME, your Groq key, and yourdomain.com.

  Test:

    docker ps                              # should show container running
    curl http://localhost:8000/health      # → {"status":"ok"}
    docker logs spending-api               # view startup logs


------------------------------------------------------------
  C7. BUILD AND DEPLOY THE FRONTEND
------------------------------------------------------------

  Still on EC2 — install Node.js:

    curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
    sudo apt install -y nodejs

  Clone the repo and build:

    cd /var/www
    sudo mkdir portfolio && sudo chown ubuntu:ubuntu portfolio
    git clone https://github.com/SouravUndefined/Portfolio.git portfolio
    cd portfolio/frontend
    echo "VITE_API_URL=https://yourdomain.com" > .env
    npm install
    npm run build

  Built site is at: /var/www/portfolio/frontend/dist/


------------------------------------------------------------
  C8. CONFIGURE NGINX
------------------------------------------------------------

    sudo nano /etc/nginx/sites-available/portfolio

  Paste (replace yourdomain.com):

    server {
        listen 80;
        server_name yourdomain.com www.yourdomain.com;

        root /var/www/portfolio/frontend/dist;
        index index.html;

        client_max_body_size 50M;

        location ~ ^/(analyse|health|files/) {
            proxy_pass         http://127.0.0.1:8000;
            proxy_set_header   Host $host;
            proxy_set_header   X-Real-IP $remote_addr;
            proxy_read_timeout 1200s;
            proxy_send_timeout 1200s;
        }

        location / {
            try_files $uri $uri/ /index.html;
        }
    }

  Save: CTRL+O → ENTER → CTRL+X

    sudo ln -s /etc/nginx/sites-available/portfolio /etc/nginx/sites-enabled/
    sudo rm -f /etc/nginx/sites-enabled/default
    sudo nginx -t
    sudo systemctl reload nginx

  Test: curl http://YOUR_ELASTIC_IP  → should return HTML


================================================================================
  PART D — CONNECT HOSTINGER DOMAIN + FREE SSL
================================================================================

------------------------------------------------------------
  D1. POINT DOMAIN TO EC2 IN HOSTINGER
------------------------------------------------------------

  1. https://hpanel.hostinger.com → Domains → your domain → DNS Records
  2. Add / edit:

       Type   Name   Value               TTL
       ----   ----   -----               ---
       A      @      YOUR_ELASTIC_IP     300
       A      www    YOUR_ELASTIC_IP     300

  3. Remove any existing A records pointing elsewhere.
  4. Wait 5-30 minutes for DNS to propagate.

  Test: ping yourdomain.com → should return your Elastic IP.


------------------------------------------------------------
  D2. GET FREE SSL CERTIFICATE
------------------------------------------------------------

  On EC2:

    sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com

  Prompts:
    - Enter email
    - Agree to terms: Y
    - Redirect HTTP to HTTPS: 2   ← always choose this

  Then:

    sudo systemctl reload nginx

  Visit https://yourdomain.com — padlock in browser.

  Auto-renewal test:

    sudo certbot renew --dry-run


------------------------------------------------------------
  D3. REBUILD FRONTEND WITH HTTPS URL
------------------------------------------------------------

    cd /var/www/portfolio/frontend
    echo "VITE_API_URL=https://yourdomain.com" > .env
    npm run build
    sudo systemctl reload nginx


================================================================================
  PART E — DEPLOYING UPDATES
================================================================================

  WHEN YOU PUSH CODE CHANGES TO GITHUB:

  ── Backend code changed ──────────────────────────────────

  On local PC — rebuild and push image:

    cd Portfolio/backend/budget-tracker
    docker build -t YOUR_DOCKERHUB_USERNAME/spending-api:latest .
    docker push YOUR_DOCKERHUB_USERNAME/spending-api:latest

  On EC2 — pull and restart:

    docker pull YOUR_DOCKERHUB_USERNAME/spending-api:latest
    docker stop spending-api
    docker rm spending-api
    # Re-run the docker run command from C6

  ── Frontend code changed ─────────────────────────────────

  On EC2:

    cd /var/www/portfolio
    git pull
    cd frontend && npm run build
    sudo systemctl reload nginx

  ── Both changed ──────────────────────────────────────────

  Do both steps above in order (backend first, then frontend).


================================================================================
  PART F — ALTERNATIVE: RENDER.COM (always free, no credit card)
================================================================================

  Use this if you don't want to manage EC2 at all.
  Trade-off: backend spins down after 15 min idle → 30-60s cold start.
  Front end on Vercel: always on, instant.

  ── F1. Deploy backend on Render ──────────────────────────

  1. https://render.com → New → Web Service
  2. Deploy an existing image from a registry
  3. Image URL: docker.io/YOUR_DOCKERHUB_USERNAME/spending-api:latest
  4. Instance Type: Free
  5. Environment Variables (add each):
       API_KEY        gsk_your_groq_key
       MODEL          meta-llama/llama-4-scout-17b-16e-instruct
       BASE_URL       https://api.groq.com/openai/v1
       FILES_DIR      /tmp/spending-files
       ALLOWED_ORIGINS *
  6. Create Web Service
  7. Your backend URL: https://spending-api-xxxx.onrender.com
  8. Test: https://spending-api-xxxx.onrender.com/health

  ── F2. Deploy frontend on Vercel ─────────────────────────

  1. https://vercel.com → New Project → Import GitHub repo
  2. Framework: Vite
  3. Root Directory: frontend
  4. Environment Variable:
       VITE_API_URL = https://spending-api-xxxx.onrender.com
  5. Deploy → your URL: https://portfolio-xxxx.vercel.app

  ── F3. Update ALLOWED_ORIGINS in Render ──────────────────

  Render → your service → Environment → update:
    ALLOWED_ORIGINS = https://portfolio-xxxx.vercel.app

  ── F4. Updating backend on Render ────────────────────────

    docker build -t YOUR_DOCKERHUB_USERNAME/spending-api:latest .
    docker push YOUR_DOCKERHUB_USERNAME/spending-api:latest

  Then on Render: Manual Deploy → Deploy latest image


================================================================================
  PART G — TROUBLESHOOTING
================================================================================

  ISSUE: docker run exits immediately
  FIX:   docker logs spending-api
         Usually a missing env var. Check API_KEY is set correctly.

  ISSUE: Nginx 502 Bad Gateway
  FIX:   docker ps — is container running?
         docker restart spending-api
         curl http://localhost:8000/health

  ISSUE: "413 Request Entity Too Large"
  FIX:   client_max_body_size 50M must be in nginx config.
         sudo nginx -t && sudo systemctl reload nginx

  ISSUE: Upload times out (Groq rate limit)
  FIX:   500K token daily limit exhausted. Wait until midnight UTC.
         Check: https://console.groq.com/ → Usage

  ISSUE: SSL cert fails / expired
  FIX:   sudo certbot renew --force-renewal
         DNS must still point to your Elastic IP.

  ISSUE: Container killed (OOM on t2.micro)
  FIX:   free -h → check swap is active (should show 2G)
         If not: sudo swapon /swapfile

  ISSUE: Downloads return 404
  FIX:   nginx location must include files/:
           location ~ ^/(analyse|health|files/) { ... }

  USEFUL COMMANDS
  ---------------
    docker ps                           # running containers
    docker logs -f spending-api         # live logs
    docker stats spending-api           # CPU + RAM usage
    docker restart spending-api         # restart without rebuild
    sudo systemctl status nginx         # nginx status
    sudo nginx -t                       # test config
    sudo systemctl reload nginx         # apply config
    df -h                               # disk usage
    free -h                             # RAM + swap
